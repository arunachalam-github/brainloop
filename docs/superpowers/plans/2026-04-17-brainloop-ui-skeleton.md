# Brainloop UI Skeleton Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Stand up the Brainloop macOS SwiftUI app skeleton — an Xcode project, a window with a top-tab navigation between four stub screens (Today / Week / Ask / Settings), and a read-only SQLite data-layer actor (`DataStore`) that can count rows in the Python daemon's existing `activity.db`. No screens show real data yet; that's Plans 2–4.

**Architecture:** SwiftUI app targeting macOS 13+. Single window, top tab bar, one `@State` tab enum drives view switching. Data layer is a GRDB-backed Swift `actor` opened read-only — the write side is still the Python daemon for now. Xcode project is regenerated from a YAML file via `xcodegen` (so `.xcodeproj` stays gitignored and the project is reproducible).

**Tech Stack:**
- Swift 5.9, SwiftUI
- macOS deployment target 13.0 (needed for `.defaultSize`, `.windowResizability`, and later `SMAppService.agent`)
- `xcodegen` for project generation ([github.com/yonaskolb/XcodeGen](https://github.com/yonaskolb/XcodeGen))
- `GRDB.swift` 6.x for SQLite (via Swift Package Manager)
- `XCTest` for unit tests

```
    │   ├── Views/
    │   │   ├── Shell/
    │   │   │   ├── RootView.swift              ← tab state + view switch
    │   │   │   └── TabBar.swift                ← small-caps serif top tab bar
    │   │   ├── TodayView.swift                 ← stub screen
    │   │   ├── WeekView.swift                  ← stub screen
    │   │   ├── AskView.swift                   ← stub screen
    │   │   └── SettingsView.swift              ← stub screen
    │   └── Data/
    │       ├── DatabasePath.swift              ← resolves DEBUG vs RELEASE path
    │       ├── ActivityRow.swift               ← minimal row model (grown per plan)
    │       └── DataStore.swift                 ← actor, read-only GRDB connection
    └── BrainloopTests/
        └── DataStoreTests.swift                ← actor smoke tests
```

**Why `.xcodeproj` is not in this tree:** it's generated by `xcodegen` from `project.yml` on demand and gitignored. Every engineer regenerates locally. This avoids the classic "two devs, two pbxproj merge conflicts" problem and makes the project definition reviewable as plain text.

**Design unit boundaries:**
- `Theme.swift` owns brand tokens; views never hardcode hex values.
- Each view in `Views/` is <100 lines; violates that → split.
- `Data/` is the only place that imports `GRDB`. Views use `DataStore` async methods.
- `DataStore` is an `actor` — all DB access is serialized, thread-safe by construction.

```swift
import SwiftUI

/// A minimal editorial tab bar: four text buttons in small-caps serif,
/// letter-spaced, with a hairline underline on the active tab.
struct TabBar: View {
    @Binding var selected: Tab

    var body: some View {
        HStack(spacing: 32) {
            ForEach(Tab.allCases) { tab in
                tabButton(tab)
            }
            Spacer()
        }
        .padding(.horizontal, 32)
        .padding(.vertical, 18)
    }

    @ViewBuilder
    private func tabButton(_ tab: Tab) -> some View {
        let isSelected = selected == tab
        Button {
            selected = tab
        } label: {
            Text(tab.label)
                .font(.system(size: 11, weight: .regular, design: .serif))
                .tracking(1.6)
                .foregroundStyle(Color.ink.opacity(isSelected ? 0.90 : 0.40))
                .padding(.bottom, 6)
                .overlay(alignment: .bottom) {
                    Rectangle()
                        .fill(Color.ink.opacity(isSelected ? 0.80 : 0))
                        .frame(height: 1)
                }
                .animation(.easeOut(duration: 0.4), value: selected)
        }
        .buttonStyle(.plain)
    }
}

#Preview {
    struct PreviewHost: View {
        @State var tab: Tab = .today
        var body: some View {
            TabBar(selected: $tab)
                .background(Color.cream)
        }
    }
    return PreviewHost()
        .frame(width: 700)
}
```

Create `app/Brainloop/Data/DatabasePath.swift`:

```swift
import Foundation

/// Resolves the absolute path to `activity.db` for the current build.
///
/// - DEBUG: reads the Python daemon's existing file in Application Support,
///   so the UI can develop against real data while the Swift daemon port is
///   still in flight.
/// - RELEASE: reads the shared app-group container at
///   `group.com.brainloop.shared` (requires the matching entitlement, added
///   in a later packaging plan).
enum DatabasePath {
    static var activityDB: URL {
        #if DEBUG
        let home = FileManager.default.homeDirectoryForCurrentUser
        return home
            .appendingPathComponent("Library")
            .appendingPathComponent("Application Support")
            .appendingPathComponent("brainloop")
            .appendingPathComponent("activity.db")
        #else
        guard let container = FileManager.default.containerURL(
            forSecurityApplicationGroupIdentifier: "group.com.brainloop.shared"
        ) else {
            fatalError("Missing app-group container — check entitlements")
        }
        return container.appendingPathComponent("activity.db")
        #endif
    }
}
```

```swift
import XCTest
import GRDB
@testable import Brainloop

final class DataStoreTests: XCTestCase {

    // MARK: - Tests

    func test_rowCount_onFreshDatabase_returnsZero() async throws {
        let url = try makeTempDatabase()
        let store = try DataStore(path: url)

        let count = try await store.rowCount()

        XCTAssertEqual(count, 0)
    }

    func test_rowCount_afterInsertingSevenRows_returnsSeven() async throws {
        let url = try makeTempDatabase()
        try insertFixtures(into: url, count: 7)
        let store = try DataStore(path: url)

        let count = try await store.rowCount()

        XCTAssertEqual(count, 7)
    }

    func test_init_onMissingFile_throws() async {
        let missing = URL(fileURLWithPath: "/tmp/definitely-not-there-\(UUID().uuidString).db")
        XCTAssertThrowsError(try DataStore(path: missing))
    }

    // MARK: - Fixtures

    /// Creates a tempfile containing a minimal `activity_log` table matching
    /// the Python daemon's schema.
    private func makeTempDatabase() throws -> URL {
        let url = URL(fileURLWithPath: NSTemporaryDirectory())
            .appendingPathComponent("brainloop-\(UUID().uuidString).db")

        let q = try DatabaseQueue(path: url.path)
        try q.write { db in
            try db.execute(sql: """
                CREATE TABLE activity_log (
                    id           INTEGER PRIMARY KEY AUTOINCREMENT,
                    ts           REAL    NOT NULL,
                    ts_iso       TEXT    NOT NULL,
                    app_name     TEXT,
                    window_title TEXT
                )
            """)
        }
        return url
    }

    private func insertFixtures(into url: URL, count: Int) throws {
        let q = try DatabaseQueue(path: url.path)
        try q.write { db in
            for i in 0..<count {
                try db.execute(sql: """
                    INSERT INTO activity_log (ts, ts_iso, app_name, window_title)
                    VALUES (?, ?, ?, ?)
                """, arguments: [
                    Double(1_776_000_000 + i),
                    "2026-04-17T00:0\(i):00",
                    "Xcode",
                    "stub #\(i)"
                ])
            }
        }
    }
}
```

```swift
import Foundation
import GRDB

/// Serializes all SQLite reads against `activity.db`. Opened read-only — a
/// belt-and-suspenders guarantee that the UI can never corrupt daemon data,
/// even through an LLM-supplied SQL query in the Ask feature (Plan 4).
///
/// The `actor` keyword gives us mutual exclusion at the Swift level; GRDB's
/// `DatabaseQueue.read` gives us its own internal queue. That's two layers of
/// serialization, which is fine — the reads are trivial and the bookkeeping
/// is simpler than trying to bypass one.
actor DataStore {
    private let dbQueue: DatabaseQueue

    init(path: URL = DatabasePath.activityDB) throws {
        var config = Configuration()
        config.readonly = true
        self.dbQueue = try DatabaseQueue(path: path.path, configuration: config)
    }

    /// Total number of rows in `activity_log`. Used by the Settings status
    /// line in Plan 2 and by the skeleton smoke tests. Not marked `async`
    /// because GRDB's `read` is synchronous — callers outside the actor
    /// still get async semantics automatically via actor isolation.
    func rowCount() throws -> Int {
        try dbQueue.read { db in
            try Int.fetchOne(db, sql: "SELECT COUNT(*) FROM activity_log") ?? 0
        }
    }
}
```
